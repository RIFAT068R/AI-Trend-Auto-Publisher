# Implementation Plan: AI Trend Auto Publisher

## Goal
Build a production-ready full-stack Next.js App Router project with a premium Apple-like glass dashboard at `/`, a history page at `/history`, reusable UI components, modular backend libraries, and robust API routes.

## Current State Snapshot
- App already uses Next.js + TypeScript + App Router and has basic `Navbar`, `Card`, pages, and API routes.
- Existing styling is extensive but currently light-themed; requested UI is dark gradient + blue/cyan glow glassmorphism.
- `Button` and `Container` reusable components are missing.
- Dashboard content currently lives in `/dashboard`; requested canonical dashboard route is `/`.
- Library modules exist and are mostly placeholders, but API response contracts are inconsistent and not fully error-safe.

## Execution Plan

### 1) Restructure Routing and Page Responsibilities
- Make `src/app/page.tsx` the canonical Dashboard page with hero, subtitle, and 3 key glass cards:
  - Latest Trends
  - Generated Post Preview
  - System Status
- Keep `src/app/history/page.tsx` as the second required page, redesigning item cards to show title, date, and status clearly.
- Keep `/dashboard` as a compatibility route by either:
  - re-exporting the root dashboard page, or
  - adding a lightweight redirect to `/`.

### 2) Build Reusable UI Primitives
- Add `src/components/Button.tsx` with typed variants (primary/secondary), glow effects, and accessible focus states.
- Add `src/components/Container.tsx` for consistent max-width and responsive spacing wrappers.
- Refine `src/components/Card.tsx` into a reusable glass panel with optional title/description/meta slots.
- Refine `src/components/Navbar.tsx` to be sticky, minimal, glassy, and aligned with routes `/` and `/history`.

### 3) Implement Premium Design System (Global CSS)
- Replace/reshape `src/app/globals.css` to match the requested visual direction:
  - Dark gradient background: `#0f172a -> #020617`
  - Glass cards: `rgba(255,255,255,0.05)` + strong backdrop blur
  - Borders: `1px solid rgba(255,255,255,0.1)`
  - Accent glow in blue/cyan
  - White / soft gray typography
- Define clean CSS variables for color, radius, blur, shadow, and spacing.
- Ensure animations are subtle and intentional (hover lift, glow intensification, reveal transitions).
- Keep responsive behavior for desktop/mobile and maintain uncluttered visual hierarchy.

### 4) Strengthen Backend Modular Architecture
- Standardize types in `src/lib/types.ts` for trends, generated preview, system status, and post history records.
- Keep and refine placeholders in:
  - `src/lib/trends.ts`
  - `src/lib/ai.ts`
  - `src/lib/image.ts`
  - `src/lib/post.ts`
  - `src/lib/storage.ts`
- Ensure each lib exports typed, isolated functions so route handlers remain thin.
- Keep file-backed storage with safe parsing/fallback behavior for `data/posts.json`.

### 5) Unify API Route Contracts and Error Safety
- Ensure all required routes exist and return typed JSON envelopes with safe `try/catch` handling:
  - `GET /api/test` -> `{ status: "working" }`
  - `GET /api/trends`
  - `GET /api/generate`
  - `GET /api/image`
  - `GET /api/post`
- Normalize responses to a consistent shape such as `{ ok, data, error }` (except `/api/test` where exact contract is required).
- Add explicit status codes for success and failure.
- Avoid leaking internal errors in responses while preserving useful server-side logs.

### 6) Data + Env Hygiene
- Keep `data/posts.json` as seeded storage for history records.
- Ensure `.env.local` usage remains optional for placeholders and does not break build/runtime without real keys.
- Keep `src/lib/env.ts` typed and non-throwing by default unless strict validation is explicitly required.

### 7) Production Readiness Checks
- Run and fix all issues from:
  - `npm run lint`
  - `npm run typecheck`
  - `npm run build`
- Perform runtime smoke checks:
  - `/` loads premium dashboard layout
  - `/history` renders glass list items with title/date/status
  - `/api/test` returns exact working payload
  - other API endpoints return valid JSON without runtime exceptions
